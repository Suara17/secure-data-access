# 🔐 安全数据访问系统 - 项目分析总结报告

## 📋 项目概述

本项目是一个基于React + TypeScript构建的企业级安全数据访问系统，实现了基于安全标记的强制访问控制(MAC)模型。系统提供了完整的用户认证、授权、数据访问控制和审计功能，适用于政府、企业等对数据安全有严格要求的场景。

## 🛠️ 技术栈分析

### 核心技术栈
- **前端框架**: React 18.3.1 with TypeScript 5.8.3
- **构建工具**: Vite 5.4.19 (现代化构建工具，支持快速热重载)
- **UI组件库**: shadcn/ui + Radix UI (基于无头组件的可访问性优先UI库)

- **路由方案**: React Router DOM 6.30.1
- **表单处理**: React Hook Form + Zod验证
- **开发工具**: ESLint + TypeScript ESLint

## 🔐 核心安全机制

### 多级安全标记系统
项目实现了**强制访问控制(MAC)**模型，包含5个安全级别：

| 安全级别 | 优先级 | 描述 |
|---------|--------|------|
| 公开 (public) | 1 | 无限制访问 |
| 内部 (internal) | 2 | 仅限组织内部访问 |
| 机密 (confidential) | 3 | 敏感信息，限制访问 |
| 秘密 (secret) | 4 | 高度敏感，严格限制 |
| 绝密 (top-secret) | 5 | 最高级别，极少数人访问 |

### 访问控制核心算法
```typescript
export const canAccess = (subjectLevel: SecurityLevel, objectLevel: SecurityLevel): boolean => {
  const subject = getSecurityLevelInfo(subjectLevel);
  const object = getSecurityLevelInfo(objectLevel);
  return subject.priority >= object.priority; // 主体级别 >= 客体级别
};
```

### 权限管理特性
1. **基于角色的访问控制(RBAC)**: admin/user角色分离
2. **数据过滤**: 根据用户安全级别自动过滤可访问数据
3. **审计日志**: 记录所有访问尝试，包括拒绝事件
4. **实时权限更新**: 管理员可动态调整用户安全级别
